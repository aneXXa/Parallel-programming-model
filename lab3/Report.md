## Задание к лабораторной работе (Task 1)

- Реализовать многопоточную версию DGEMV с параллельной инициализацией массивов.
- Провести измерения для 1,2,4,7,8,16,20,40 потоков.
- Использовать размеры матриц 20000x20000 и 40000x40000.
- Построить графики ускорения и эффективности, сделать вывод о масштабируемости.

## Цель

Реализовать параллельное умножение матрицы на вектор (тип `double`) с параллельной инициализацией массивов, измерить масштабируемость для 1,2,4,7,8,16,20,40 потоков на размерах 20000x20000 и 40000x40000, и сделать выводы, подтвержденные численными результатами.

## Методика

- Последовательная версия: `matrix_vector_product`.
- Параллельная версия: `matrix_vector_product_omp`.
- Инициализация данных: `init_arrays_parallel`.
- Замер времени: несколько запусков, отброс максимального значения, усреднение оставшихся.
- Метрики:
  - `T_serial, T_n`
  - `S_n = T_serial / T_n`
  - `E_n = S_n / n`

## Гипотезы

1. Для обоих размеров ускорение растет при увеличении числа потоков, но не монотонно на больших `n`.
2. Оптимальная точка для `40000x40000` в данном запуске — около 16 потоков.
3. Эффективность падает с ростом числа потоков, особенно после выхода за комфортное число потоков.
4. Почему ускорение падает при 40 потоках (`40000x40000`):
   - 12.8 GiB не помещается в кэш, потоки конкурируют за пропускную способность памяти.
   - При 40 потоках используются оба сокета; доступ к памяти удаленной NUMA-ноды медленнее.
   - Миграция потоков между сокетами может усиливать NUMA-накладные расходы.
