# Лабораторная работа 2 — Task 3 (метод простой итерации, OpenMP)

## Цель

Сравнить две OpenMP-реализации метода простой итерации для `Ax=b`:

1. Вариант 1 (`blocks`) — отдельный `#pragma omp parallel for` для каждого распараллеливаемого цикла.
2. Вариант 2 (`whole`) — одна общая область `#pragma omp parallel` на весь итерационный алгоритм.

Провести анализ времени, ускорения и эффективности.

## Постановка задачи

- `A[i][i] = 2.0`, `A[i][j] = 1.0` при `i != j`.
- `b[i] = N + 1`.
- Точное решение: `x[i] = 1.0`.
- Начальное приближение: `x = 0`.

## Методика

- База: `solve_jacobi_serial`.
- Параллельные версии:
  - `solve_jacobi_parallel_blocks`;
  - `solve_jacobi_parallel_whole`.
- Замеры: многократные запуски, отброс максимума, усреднение.
- Метрики:
  - `T_blocks`, `T_whole`;
  - `S_blocks`, `S_whole`;
  - `E_blocks`, `E_whole`.

## Выводы с доказательствами

1. **Обе версии ускоряются с ростом числа потоков, но далеки от линейного масштабирования.**  
   - `blocks`: `S_1 = 0.149` → `S_40 = 2.481`;  
   - `whole`: `S_1 = 0.148` → `S_40 = 2.292`.

2. **По скорости варианты близки; на 16–20 потоках чуть лучше `whole`, на 40 потоках лучше `blocks`.**  
   - 16 потоков: `T_whole = 1.466` против `T_blocks = 1.501`;  
   - 20 потоков: `T_whole = 1.293` против `T_blocks = 1.334`;  
   - 40 потоков: `T_blocks = 1.112` против `T_whole = 1.204`.

3. **Эффективность снижается при увеличении числа потоков (типичный эффект насыщения).**  
   - `blocks`: `E_1 = 0.149` → `E_40 = 0.062`;  
   - `whole`: `E_1 = 0.148` → `E_40 = 0.057`.

4. **На данном наборе параметров ни один вариант не даёт радикального преимущества по всем точкам.**  
   - Кривые `S_blocks` и `S_whole` в `task3_all_graphs_6000.pdf` идут близко; локальные преимущества меняются с числом потоков.

Итог: практический выбор варианта нужно делать по целевому диапазону потоков и требуемому времени. Для 16–20 потоков в этом измерении лучше `whole`, для 40 — `blocks`.
